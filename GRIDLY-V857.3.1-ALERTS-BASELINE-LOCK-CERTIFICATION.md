# Gridly V857.3.1 — Alerts Baseline Lock Certification

## Certification declaration

**ALERTS BASELINE V1**

**Status: LOCKED**

Future Alerts presentation changes should require at least one of the following:

- a demonstrated usability issue;
- user testing evidence;
- a new product requirement.

Aesthetic-only Alerts refinements should be avoided after this milestone.

## Certification findings

- Alerts now uses the consumer-facing sheet title **Current Alerts** where the portrait Alerts surface is opened.
- Multi-line alert headlines remain a single visual statement. Hierarchy is handled with title weight and line-height rather than extra spacing.
- Supporting metadata remains present, but its visual weight is reduced so lines such as report freshness, community attribution, and confidence context read as secondary trust information.
- Alert cards keep the Home Screen/Search baseline card language: rounded corners, quiet border, restrained shadow, and premium low-noise contrast.
- Portrait scrolling remains contained with a slightly clearer scrollbar treatment.
- The empty state wording remains unchanged and continues to read calm, confident, and reassuring.

## Protected-system confirmation

This certification pass is presentation-only. It does not change alert generation, hazard lifecycle, community reports, unified intelligence, search, route generation, Route Watch, awareness, weather providers, Supabase, data providers, filtering logic, runtime architecture, or provider behavior.

## Merge recommendation

Merge is recommended after validation. Home Screen Baseline V1, Search Baseline V1, and Alerts Baseline V1 now define the primary Gridly visual language.
