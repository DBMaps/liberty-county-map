# LP036.1C Runtime County Geometry Package Tooling

LP036.1C is an infrastructure-only recovery milestone. The combined generated runtime county geometry package is intentionally **not committed** in this Codex recovery task because the package is a large generated geometry artifact and exceeded Codex Cloud extraction limits in the previous attempt.

## Local build commands

Build the package locally after checkout:

```bash
node tools/build-gridly-authoritative-county-geometry.js
```

Verify deterministic serialization from unchanged source files:

```bash
node tools/build-gridly-authoritative-county-geometry.js --verify-deterministic
```

## Generated output paths

The builder writes exactly these generated files:

- `assets/location-resolution/gridly-authoritative-county-geometry-v1.json`
- `assets/location-resolution/gridly-authoritative-county-geometry-v1.manifest.json`

The package file remains generated output and should not be committed as part of this recovery task.

## County requirement

The builder reads the authoritative `GRIDLY_COUNTY_REGISTRY` from `js/app.js`, uses each operational county's registered `boundaryPath`, and requires exactly 28 operational counties.

## Certification expectations

The build-time point-in-polygon certification includes these required checks:

- Dayton (`30.0466, -94.8852`) must resolve inside `liberty-tx` geometry.
- Livingston (`30.7110, -94.9327`) must resolve inside `polk-tx` geometry.
- Houston and Pasadena must resolve inside `harris-tx` geometry.
- An outside-coverage point must resolve to no operational county.
- Boundary handling, Polygon support, MultiPolygon support, and polygon-hole support are verified deterministically.

## Browser audit command

After running the local builder and serving the app, run this in the browser console:

```js
await window.gridlyLp0361cRuntimeCountyGeometryPackageAudit?.()
```

The audit distinguishes configuration readiness from generated package presence and certification. It must not report LP036.1D readiness until the generated package and manifest are physically present and the certification metadata passes.

## Deferred production integration

LP036.1C does **not** integrate polygon containment into the LP036.1 passive resolver and does **not** change current production county resolution behavior. Production resolver integration remains deferred to LP036.1D. LP036.2 readiness remains false.
