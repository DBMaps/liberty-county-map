# LP037.2 — Regional Consumer Crossing Count Repair

## Shared selector

LP037.2 establishes `gridlySelectConsumerVisibleCrossings()` as the passive, shared selector for consumer crossing counts. The selector starts from the certified active-county runtime crossing inventory, applies the existing production visibility policy, applies reportability, checks awareness-area geographic ownership, and removes duplicate FRA IDs before returning consumer-visible crossings.

## Migration

Consumer-facing count ownership now routes through the shared selector instead of independent county, town, radius, or bottom-panel calculations. `getGridlyHomeTownCrossings()` is a compatibility wrapper over the selector so existing Awareness Brief, Community Pulse, crossing copy, and audit consumers share the same count source.

## Consumer ownership

The consumer count owner is `gridlySelectConsumerVisibleCrossings`. Marker rendering, viewport optimization, runtime inventory loading, crossing packages, county ownership, roadway runtime, reporting, alerts, route watch, and provider behavior are intentionally unchanged.

## Regional certification

The passive browser audit `window.gridlyLp0372ConsumerCrossingCountRepairAudit?.()` certifies operational county-mode areas, configured community-mode areas, and Houston child regions against the selector. Certification verifies selector presence, duplicate-FRA removal, production visibility application, awareness geometry application, and shared consumer ownership.

## Spring Branch verification

No Spring Branch-specific logic was added. If Spring Branch is configured as an awareness area, it is certified by the same community-mode selector path as every other configured community. If it is not configured in the current runtime definitions, the audit reports that absence rather than inventing an override.

## Future maintenance

Future consumer crossing count surfaces must call `gridlySelectConsumerVisibleCrossings()` or a wrapper that delegates to it. Do not introduce surface-local crossing counts for individual counties, communities, Houston regions, or Spring Branch. Any future change to visibility policy or reportability should remain in its existing policy owner and be consumed by the selector.
