# LP038.2 — Weather Authority Source Integration

## Existing source path

LP038.2 keeps the existing weather runtime path intact:

1. `gridlyWeatherProvider` owns provider identity, NWS payload validation, raw NWS feature extraction, and normalized weather alert records.
2. `gridlyWeatherLiveConnector` owns the existing NWS active-alert endpoint, fetch lifecycle, timeout/retry behavior, connector-retained normalized records, awareness filtering, expiration retention behavior, and official-provider refresh notification.
3. `gridlySelectConsumerWeatherAuthority()` owns the LP038.1 canonical authority result, consumer eligibility, geographic ownership, freshness ownership, deduplication, and quiet-state reasoning.
4. Existing consumer surfaces continue reading their current owners. LP038.2 does not migrate Awareness Brief, Community Pulse, Travel Brief, weather cards, map weather presentation, alert cards, count copy, quiet-state copy, or consumer summaries.

## Integration boundary

`js/gridlyWeatherAuthoritySourceIntegration.js` is a passive source adapter and snapshot layer. It observes already-loaded normalized weather records from the connector or provider, adapts them to the LP038.1 authority selector contract, and returns a non-UI authority snapshot. It does not fetch, poll, render, write storage, move the map, mutate selected awareness state, or start connector activation.

## Source adapter contract

`gridlyAdaptWeatherRecordsForAuthority(records, options)` preserves source evidence including provider identity, provider record ID, event ID, headline, event/category, severity, effective/onset/expiration/sent timestamps, geometry, alert polygon, zone references, county references, area description, normalized coordinates, source locality text, travel-impact classification, and connector-retained state.

The adapter reports raw, unique, duplicate, expired, stale, missing-timestamp, and future-effective counts. Stable provider/source IDs are preferred for deduplication; records without stable IDs use a deterministic fallback key built from provider, event, timestamps, and area evidence.

## NWS alert integration

The production NWS alert integration uses existing normalized records. The source integration does not duplicate NWS fetches or normalization. The intended flow is:

NWS provider records → existing Gridly normalization → source integration adapter → `gridlySelectConsumerWeatherAuthority()` → canonical weather authority result.

## Ownership hierarchy

Authority ownership follows the strongest available source evidence:

1. Alert polygon or geometry intersection.
2. Certified zone ownership.
3. Certified county ownership.
4. Selected-awareness geometry or configured awareness anchor relationship.
5. Existing connector radius/text evidence as a disclosed fallback.

Fallback text/locality ownership is exposed as fallback evidence and is not promoted as polygon-certified ownership.

## Fallback disclosure

Eligible authority records disclose ownership method, ownership confidence, fallback usage, fallback reason, whether source geography was available, source geography type, and selected-awareness match. Missing geography remains uncertain; LP038.2 does not invent polygons, zones, counties, or locality.

## Freshness contract

The adapter distinguishes active, expired, stale, future-effective, and missing-timestamp source records. Expired records are excluded from the authority-eligible path. Stale, future-effective, and missing-timestamp records are disclosed and are not presented as fresh active weather. Missing data is not classified as clear weather.

## Deduplication contract

Source-level deduplication is part of the authority path. The snapshot reports `rawRecordCount`, `uniqueProviderRecordCount`, `duplicateRecordCount`, `expiredRecordCount`, `authorityEligibleRecordCount`, and `uniqueSituationCount`.

## Current-conditions status

LP038.2 does not add or activate a current-conditions provider. The source integration status reports current conditions as unavailable and unintegrated unless a real active provider is later discovered by a future milestone. No temperature, wind, visibility, precipitation, station, distance, or locality values are fabricated.

## Forecast status

LP038.2 does not add or activate a forecast provider. Point, grid, and zone forecast adapter presence is declared as a passive contract only, while forecast source availability and integration remain false unless a real active provider exists in a later milestone. No point forecast, grid forecast, forecast zone, or county forecast is fabricated.

## Authority snapshot

`gridlyGetWeatherAuthoritySnapshot(options)` returns the latest passive authority result for the selected awareness area, active county/community context, loaded source records, source integration status, freshness status, geographic ownership status, deduplication status, and quiet-state cause. The snapshot is read-only and non-UI.

## Browser audit helper

`window.gridlyLp0382WeatherAuthoritySourceIntegrationAudit?.()` verifies foundation presence, selector presence, source adapter presence, authority snapshot presence, NWS provider/normalization presence, alert integration, current-conditions/forecast source truth, geographic ownership, freshness, deduplication, consumer eligibility, no polling, no fetches, no writes, no storage writes, no map movement, no UI migration, and LP038.3 readiness.

## Consumer migration boundary

Consumer UI migration is explicitly out of scope. Existing production weather behavior remains visually and functionally unchanged. LP038.2 creates source authority plumbing only.

## Known limitations

- Alert ownership quality depends on the fields retained by existing normalized records.
- Zone certification is only used when source records and selected-awareness configuration expose matching zone identifiers.
- County/text fallback remains lower-confidence disclosed evidence.
- Current conditions and forecasts are contracts only; no source provider is activated in this milestone.

## LP038.3 readiness

LP038.2 prepares LP038.3 to migrate consumer weather surfaces to the shared authority snapshot without changing provider activation, NWS endpoints, polling cadence, or consumer rendering in this milestone.
