# Gridly Beta Release Checklist

## Purpose

This is Denise's release checklist for preparing, validating, and approving Gridly beta distribution packages.

## Release Checklist

### Documentation Review

- [ ] Beta landing page information is current.
- [ ] Install guidance is accurate for supported devices.
- [ ] What's New content is written in consumer language.
- [ ] Support documentation reflects current known limitations.

### Visual Review

- [ ] Mobile portrait remains the primary review target.
- [ ] Landing and support materials feel premium, simple, and trustworthy.
- [ ] Tester-facing copy is readable on a phone.

### Audit Review

- [ ] Certification document is complete.
- [ ] Runtime and protected-system statements are explicit.
- [ ] Known limitations are documented.

### Install Review

- [ ] Current beta URL opens successfully.
- [ ] Install or add-to-home-screen guidance is accurate.
- [ ] Old shortcut recovery guidance is available.

### Real Device Verification

- [ ] iPhone browser review completed.
- [ ] Android browser review completed.
- [ ] Desktop support preview completed if needed.

### Known Issues Updated

- [ ] Known issues are current.
- [ ] Workarounds are included where available.
- [ ] Safety-critical limitations are clear.

### What's New Written

- [ ] Highlights are concise.
- [ ] Known issues are included.
- [ ] Resolved issues are included when applicable.
- [ ] Coming soon language does not overpromise.

### Support Documentation Updated

- [ ] FAQ is current.
- [ ] Escalation guidance is current.
- [ ] Feedback workflow is clear.

### Version Updated

- [ ] Milestone version is correct.
- [ ] Distribution URL version is correct.
- [ ] Documentation references the current version.

### Beta URL Verified

- [ ] Current beta URL is listed exactly.
- [ ] Invite copy uses the current beta URL.
- [ ] Retired URLs are no longer being shared.

### Invite Communication Prepared

- [ ] Tester invite copy is ready.
- [ ] Feedback instructions are included.
- [ ] Official-source disclaimer is included.

### Rollback Plan Available

- [ ] Previous stable beta URL is known if rollback is needed.
- [ ] Tester communication plan is ready.
- [ ] Support team knows how to identify affected testers.

### Release Approval

- [ ] Documentation owner approval.
- [ ] Product owner approval.
- [ ] Beta distribution approval.
- [ ] Merge recommendation recorded.

## Success Criteria

The release is ready when all required checklist items are complete or explicitly waived by the release owner with a reason.
