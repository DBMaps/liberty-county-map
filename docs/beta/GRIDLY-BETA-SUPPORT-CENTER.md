# Gridly Beta Support Center

## Purpose

This document provides a lightweight support strategy for Gridly beta testers. It focuses on clear answers, fast triage, and calm guidance without creating a heavy support operation.

## Support Philosophy

Gridly support should be:

- Friendly and direct
- Awareness-first
- Honest about beta limitations
- Respectful of tester time
- Careful not to replace official sources
- Focused on repeatable issue collection

## Frequently Asked Questions

### Installation

**Do I need to install Gridly?**  
No. Testers can use the beta link in a browser. Installing or adding to the home screen is optional.

**Why do I not see an install button?**  
Install options vary by browser and device. Use the browser menu when available or continue in the browser.

**How do I fix an old shortcut?**  
Remove the old shortcut, open the current beta URL, then add the page again.

### Reporting

**What should I report?**  
Report road, weather, or local awareness conditions only when it is safe and appropriate.

**What should I not report?**  
Do not submit private personal information, speculation presented as fact, or anything that would distract from safe driving.

**Are reports reviewed?**  
Beta feedback and reports may be reviewed to improve the experience, but community reports should not be treated as official verification.

### Awareness

**What is the Awareness Brief?**  
It is a quick summary intended to help testers understand nearby conditions before leaving.

**Is Gridly a navigation replacement?**  
No. Gridly is awareness first and route intelligence second.

### Community Reports

**Can community reports be wrong?**  
Yes. Community reports can be incomplete, delayed, duplicated, or inaccurate.

**How should I use Community Pulse?**  
Use it as context about local activity, not as a guarantee or official source.

### Privacy

**Should I include personal information in a report?**  
No. Reports should avoid private personal information.

**How is feedback used?**  
Feedback may be reviewed by the Gridly team to improve beta quality, communication, and support.

### Location

**Do I have to allow location?**  
No. Location improves local awareness but is optional.

**What if I denied location by mistake?**  
Update browser or device permissions, reload the beta, and try again.

### Known Limitations

Gridly beta may include incomplete data, stale information, delayed updates, device-specific install differences, and unverified community reports. It is not an emergency notification service.

## Escalation Guidance

Escalate feedback when it involves:

- Safety-critical confusion
- Repeated install failure across multiple testers
- Incorrect or misleading beta instructions
- Privacy concerns
- Official-source confusion
- Major device-specific blockers

Immediate safety issues should be directed to official emergency or public-safety channels first.

## Support Workflow

1. Receive the issue.
2. Confirm device, browser, time, and current beta URL.
3. Identify whether it is install, onboarding, awareness, reporting, privacy, or support-copy related.
4. Check whether it is already a known limitation.
5. Provide a simple workaround if available.
6. Record unresolved issues for triage.
7. Update support documentation when the same confusion repeats.

## Success Criteria

The support center is successful when:

- Testers can self-solve common install and permission issues.
- Support responses stay consistent.
- Feedback includes enough detail for triage.
- Community and official-source disclaimers are clear.
- Support remains lightweight and sustainable during beta expansion.
