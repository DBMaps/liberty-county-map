# Gridly Mobile Install Guide

## Purpose

This guide explains how mobile beta testers should open, install, launch, update, and remove Gridly on supported mobile devices. It does not reference desktop installation because desktop remains intentionally gated.

## Open the Beta URL

Open the current beta URL on a phone in portrait orientation:

<https://dbmaps.github.io/liberty-county-map/?v=beta-clean-4>

Expected result: Gridly opens in the mobile browser and presents the supported Mobile Portrait beta experience.

## Install from Browser

Installing Gridly means adding the beta web app to the phone home screen using the phone browser. This makes repeat access easier, but testers may also continue using the browser link.

## iPhone Instructions

1. Open the beta URL in Safari.
2. Keep the phone in portrait orientation.
3. Tap the Share button.
4. Choose **Add to Home Screen**.
5. Confirm the name is Gridly.
6. Tap **Add**.
7. Open Gridly from the new home-screen icon.

## Android Instructions

1. Open the beta URL in Chrome or the preferred supported browser.
2. Keep the phone in portrait orientation.
3. Open the browser menu.
4. Choose **Install app** or **Add to Home screen** when available.
5. Confirm the name is Gridly.
6. Tap **Install** or **Add**.
7. Open Gridly from the new home-screen icon.

## Launching Gridly

Launch Gridly from either:

- The current beta URL in the mobile browser.
- The Gridly home-screen icon created during install.

If the icon opens an old or unexpected version, remove it and reinstall from the current beta URL.

## Completing Onboarding

On first launch, complete the onboarding flow. The onboarding should explain Gridly's mission, beta status, location awareness, and community-awareness purpose.

## Selecting Awareness Area

Choose or confirm the awareness area requested by the app. During V902 beta, testers should treat Liberty County as the current county focus unless instructed otherwise by the rollout owner.

## Granting Location

If prompted, grant location only if comfortable. Location helps Gridly understand nearby awareness context. If a tester declines location, they should note whether the app still explains what to do next.

## First Launch Expectations

On first launch, testers should expect to:

- See a Mobile Portrait experience.
- Complete onboarding if it has not already been completed.
- Confirm or select an awareness area.
- Review the map and Awareness Brief.
- Understand that Gridly is beta and not for emergencies.

## Updating Gridly

To update Gridly during beta:

1. Close Gridly.
2. Reopen the current beta URL.
3. Refresh the browser page if needed.
4. Remove and reinstall the home-screen icon if it continues opening an old version.

## Removing Gridly

To remove Gridly, delete the home-screen icon using the phone's normal remove-app or remove-shortcut action. This does not require desktop steps.

## Common Problems

### Gridly opens the wrong version

Reopen the current beta URL and reinstall the home-screen icon.

### Add to Home Screen is hard to find

Confirm the tester is using the mobile browser menu or Share sheet. On iPhone, use Safari. On Android, use the browser menu.

### Location prompt was declined

Continue testing and note whether the app provides a clear path forward. If needed, re-enable location from phone browser or site settings.

### App looks rotated or unsupported

Return the phone to portrait orientation. Landscape remains intentionally gated.

### Page does not load

Check connection, close and reopen the browser, then open the current beta URL again.

## Recovery Guidance

If installation or launch fails:

1. Confirm the phone is in portrait orientation.
2. Confirm the current beta URL was used.
3. Close and reopen the browser.
4. Refresh the page.
5. Remove any old Gridly home-screen icon.
6. Reinstall from the current beta URL.
7. Send feedback with device, browser, issue, and screenshot if possible.

## Success Criteria

The install experience succeeds when testers can open the current beta URL, optionally install Gridly, launch it again, complete onboarding, select an awareness area, understand first-launch expectations, and recover from common mobile install issues without desktop instructions.
