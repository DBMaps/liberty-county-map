# Gridly Mobile Screenshot Standards

## Purpose

This standard keeps Gridly beta screenshots consistent, safe, and aligned with the supported Mobile Portrait experience.

## Orientation

Use portrait orientation only. Do not capture landscape screenshots.

## Platform Scope

Do not capture desktop screenshots for V902 beta materials. Desktop remains intentionally gated, and screenshots should not imply desktop support.

## Branding

Use production Gridly branding. Avoid draft overlays, unrelated icons, debug labels, or edited branding unless a review explicitly requires annotation.

## Personal Information

Hide personal information before sharing screenshots. Do not expose addresses, precise home locations, phone notifications, names, contact details, or private browser information.

## Capture Full Screen

Capture the full phone screen unless a specific issue requires a cropped detail. Full-screen captures help reviewers evaluate spacing, safe areas, browser chrome, install mode, and supported orientation.

## Preferred Devices

Use current iPhone and Android devices representative of tester phones. Prioritize common screen sizes and avoid relying only on unusually large or unusually small devices.

## Screenshot Naming

Use clear names:

`gridly-v902-mobile-portrait-[screen]-[device]-[date].png`

Examples:

- `gridly-v902-mobile-portrait-onboarding-iphone-2026-07-06.png`
- `gridly-v902-mobile-portrait-settings-android-2026-07-06.png`

## Version Labeling

When possible, include the beta version or beta URL in the screenshot record, caption, or accompanying note. Do not add misleading labels directly onto screenshots unless the image is clearly marked as annotated.

## When Screenshots Should Be Updated

Update screenshots when:

- The beta URL or version label changes.
- Onboarding wording changes.
- Install guidance changes.
- Mobile Portrait layout changes.
- Settings, Awareness Brief, Community Pulse, or reporting screens materially change.
- Branding changes.
- A screenshot exposes outdated limitations, unsupported platforms, or personal information.
